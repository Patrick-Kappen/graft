# Runtime architectuur en cleanup

Deze notitie legt de runtime-basis vast. Zie ook [`vision.md`](vision.md) voor het hogere doel: `pac` als direnv voor containers met enter/leave/idle lifecycle.

## Huidige werkende slice

De eerste CLI-slice doet conceptueel:

```text
CLI
  -> tijdelijke rootfs directory
  -> tijdelijke Quadlet .container in $XDG_RUNTIME_DIR/containers/systemd
  -> systemctl --user daemon-reload/start
  -> Podman draait met Rootfs=
  -> logs tonen
  -> tijdelijke files opruimen
```

Er wordt geen `Image=` gebruikt. De container gebruikt:

```text
Rootfs=<temp-rootfs>
Volume=/nix/store:/nix/store:ro
Exec=<absolute /nix/store/... command>
```

Dit bewijst alleen dat de runtime-keten werkt:

```text
Quadlet -> systemd --user -> Podman -> rootfs-store command
```

## Gewenste flow

De flow is:

```text
TOML
  -> load
  -> normalize
  -> resolve graph
  -> validate
  -> effective config
  -> render Quadlet
  -> apply/run via systemd
  -> cleanup volgens lifecycle policy
```

## Scheiding van domeinen

De implementatie moet twee kernmodules krijgen:

```text
config engine
  TOML laden, normaliseren, graph resolven, mergen, valideren

runtime manager
  Quadlet files schrijven, systemd reload/start/stop, logs tonen, cleanup
```

`run-rootfs` is tijdelijk één gebruiker van de runtime manager. Later moet `run <entry>` dezelfde runtime manager aanroepen op basis van effective TOML config.

## Minimale TOML-run

Eerste TOML-gedreven stap mag nog zonder graph zijn:

```toml
version = 1
name = "hello"

[config.runtime]
mode = "rootfs-store"
packages = ["bashInteractive", "coreutils"]
command = ["bash", "-lc", "echo hello from toml"]
```

Beschikbare commands:

```bash
pac render ./examples/rootfs-store.toml
pac run ./examples/rootfs-store.toml
pac up ./examples/rootfs-store.toml
```

Daarna pas graph resolving en project sessions toevoegen:

```bash
pac up
pac enter
pac leave
```

## Lifecycle: transient versus managed

Er zijn twee hoofdmodi.

### Transient run

Voor agent-runs, smoke tests en one-shot jobs.

```text
Quadlet location: $XDG_RUNTIME_DIR/containers/systemd
rootfs/tmp:        $XDG_RUNTIME_DIR/podman-agent-container/runs/<run-id>
cleanup:           standaard na exit
```

### Managed service

Voor echte langdurige containers.

```text
Quadlet location: ~/.config/containers/systemd
state:            persistent tenzij expliciet verwijderd
cleanup:          alleen via apply/remove/cleanup command
```

Later kan system-level beheer onder `/etc/containers/systemd` worden toegevoegd, maar de eerste implementatie blijft rootless user-mode.

## Cleanup-objecten

De tool moet expliciet nadenken over wat tijdelijk of persistent is.

### Tijdelijke Quadlet files

```text
$XDG_RUNTIME_DIR/containers/systemd/*.container
```

Transient runs ruimen deze na afloop op.

### Tijdelijke rootfs/run directories

```text
$XDG_RUNTIME_DIR/podman-agent-container/runs/<run-id>/
```

Standaard opruimen na exit. Een debug/keep mode kan later toegevoegd worden.

### Podman containers

Quadlet/systemd genereert normaal `podman rm` bij stop als `--rm` gebruikt wordt. Toch moet de tool stale containers kunnen opsporen.

### Volumes en state

Volumes mogen niet impliciet weggegooid worden. Per volume moet later een lifecycle mogelijk zijn:

```toml
volumes:
  - name: agent-home
    lifecycle: transient
  - name: cache
    lifecycle: persistent
```

### Logs

Journald logs blijven normaal bewaard. De tool toont logs via:

```bash
journalctl --user -u <unit>
```

Logs worden niet actief verwijderd door de eerste cleanup-flow.

## Labels en naming

Alle gegenereerde resources moeten herkenbaar zijn als eigendom van deze tool.

Naamconventie:

```text
pac-<entry-or-kind>-<run-id>
```

Podman labels:

```ini
Label=io.github.zerodawn.podman-agent-container=true
Label=io.github.zerodawn.podman-agent-container.entry=<entry>
Label=io.github.zerodawn.podman-agent-container.run-id=<run-id>
Label=io.github.zerodawn.podman-agent-container.lifecycle=transient|persistent
```

Quadlet header:

```ini
# Generated by podman-agent-container
# entry=<entry>
# run-id=<run-id>
# lifecycle=transient|persistent
```

Cleanup mag alleen resources verwijderen die zo gemarkeerd zijn.

## Gewenste commands

Eerst:

```bash
podman-agent-container render <toml-or-entry>
podman-agent-container run <toml-or-entry>
podman-agent-container cleanup
```

Later:

```bash
podman-agent-container cleanup --older-than 24h
podman-agent-container cleanup --all-transient
podman-agent-container ps
podman-agent-container status <entry>
podman-agent-container apply <entry>
podman-agent-container remove <entry>
```

## Implementatievolgorde

1. Houd de bestaande rootfs/Quadlet slice als runtime basis.
2. Voeg TOML laden toe voor één bestand, nog zonder graph.
3. Implementeer `render <file>` naar Quadlet text.
4. Implementeer `run <file>` via dezelfde runtime manager.
5. Voeg labels, headers en lifecycle metadata toe.
6. Voeg `cleanup` toe voor stale transient resources.
7. Voeg daarna pas TOML graph resolving toe.
