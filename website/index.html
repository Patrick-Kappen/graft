Why Graft

Container intent without
the hidden machinery.

Graft keeps the workload definition small while making every materialisation boundary explicit and reviewable.

01

Typed TOML intent

Workloads declare packages, commands, and selected container policy without embedding Nix modules or raw Quadlet maps.

Read the reference →
02

Nix-built rootfs

Packages resolve from the target host's pinned nixpkgs. The running container performs no package installation.

Explore the architecture →
03

Systemd-native runtime

Quadlet generates ordinary systemd services. Podman runs the container; Graft does not introduce another supervisor.

Inspect the output →

The current path is real—and deliberately narrow.

Graft currently supports NixOS system containers and Home Manager user containers through the same typed resolver. Secure defaults, lifecycle commands, temporary instances, and multi-host control remain active roadmap work.
